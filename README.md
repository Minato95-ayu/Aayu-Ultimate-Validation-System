## 🚀 Introduction

**Aayu** is a powerful, custom-built, English-like programming language created by **Ayush Kaushik**. It is designed to be highly readable, intuitive, and extremely modular. 

The **Aayu Ultimate Validation System (AUVS)** is the official testing and validation suite for Aayu v1. Its primary goal is to aggressively test and prove all core features, subsystems, and the architecture of the language in a single, unified environment.

---

## ✨ Features of Aayu Language

Aayu comes fully loaded with modern programming paradigms wrapped in a simple, natural-language syntax:

- **Readable Syntax:** Code that reads like plain English (e.g., `number a is 10.`, `show "Hello".`).
- **Data Structures (Records):** Object-oriented data modeling using `record`.
- **Functions (Tasks):** Defining and executing tasks using `task` and `run`.
- **Control Flow:** `if / else`, `while` loops, `repeat x times`, and `for each`.
- **File Persistence (I/O):** Built-in `read` and `write` functionality for file handling.
- **Error Handling:** Safe execution with `try` and `catch` blocks.
- **Modularity:** Multi-file project support using the `use` keyword.
- **Intent Engine (AI):** A revolutionary engine that converts natural language intents into fully working Aayu source code and executes it instantly!

## 🛠️ Validation Phases (AUVS)

The Ultimate Validation System proves Aayu's stability through 6 rigorous phases. **All phases are 100% PASS.**

### 🟢 Phase-1: Records Compilation
Validates the language's ability to parse and compile complex data models (`user.aayu`, `student.aayu`, `library.aayu`, `book.aayu`, `post.aayu`) with zero syntax errors.

### 🟢 Phase-2: Database Layer (CRUD)
Validates file-based persistence. It tests the ability to `Insert`, `Search`, `Update`, and `Delete` records dynamically using Aayu's built-in file I/O.

### 🟢 Phase-3: API Layer
Validates multi-file architecture where `api.aayu` successfully imports and calls functions (tasks) from `database.aayu` simulating `POST`, `GET`, `PUT`, and `DELETE` requests.

### 🟢 Phase-4 & Phase-5: The Main Runner
The central `main.aayu` and `validation.aayu` runner that executes all the interconnected modules and generates the final validation report.

### 🟢 Phase-6: Intent Engine Validation
Tests Aayu's built-in **Intent Engine**. It proves that the engine can successfully parse valid intents (like *"Create a Library Management System"*), generate the corresponding Aayu architecture graph, write the code, and execute it. It also correctly rejects invalid intents (like *"Create a Nuclear Reactor"*).

## 💻 How to Run AUVS

To run the ultimate validation report, ensure you have Python installed, and execute the main runner:

```bash
python run.py main.aayu
```

To run the Intent Engine validation tests:

```bash
python phase6_test.py
```
